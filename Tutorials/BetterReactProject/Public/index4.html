Lesson 4

Now we are going to create webpack config file so that we dont have to run the entire command webpack ./public/app.js ./public/bundle.js for every change in the app.js to convert to bundle.js and also the code we are using in the bundle.js is still readable so to avoid this code visibility we are learning this step

  1. In the main project folder create a file with the exact name as webpack.config.js
  2. add lines module.exports
  3. and add entry: the source file path
  4. output: {path: __dirname [main folder/project path this can be given by "__dirname" in node], filename:'path after the main directory[./public/...]'}
  5. { extensions: ['','.js','.jsx']} these are the file extensions to be considered or resolved
  6. Now the changes in the app.js can be added in bundle.js with single command that is webpack this calls the webpack.config.js file and this executes the command webpack ./public/app.js ./public/bundle.js


Lesson 5

Now we are using the app.jsx file instead of app.js for this we gonna need babel-loader package which we have installed already to use this in webpack.config.js file we have to follow the following steps

  1. Add modules: { loaders: [ { loader: 'babel-loader', query:{presets:['react', 'es2015']}, test: /\.jsx?$/. } ] } Here we are telling the babel loader to take our files and parse them through react and get output and run them through es2015
  2. the files to get are provided bt using the test attribute to set the file
    1. we are using hte file extension as .jsx here, by regular expresion matching $ : says the extension is considered at the end of the file name i.e., match the end of the file name, \. : escape character as . is special character in regular expression
  3. We also have to restrict some folder to not get searched by the test attribute for this we have to use exclude attribute and set the folders to exclude as exclude: /(node_modules|bower_components)/
  4. while using webpack we should not usr the react and react-dom in .jsx as a global components as we used then till now instead we have to use require(''); method and include into the .jsx file and we have to use that object
